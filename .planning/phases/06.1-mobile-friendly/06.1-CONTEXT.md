# Phase 6.1: Mobile Friendly (INSERTED) - Context (design brief)

**Gathered:** 2026-07-06 · **Status:** Ready for execution
**Trigger:** User: "Make it also mobile friendly please."

<diagnosis>
## Live mobile audit (390×844, iPhone-13 UA, real Playwright captures)

What already works: viewport meta present; no horizontal scroll; node TAP picking works (hit-proxy); WebGL renders; zero console errors; DetailPanel already goes full-width via its existing media query.

What is broken (from the captures):
1. **Top collision** — SceneTitle (fixed TL) and PipelineReadout (fixed TR) overlap each other on 390px; "$20,000" renders on top of "GRANT CRYSTARIUM".
2. **Bottom ghosting** — the Legend (fixed BL) renders THROUGH the PipelineDrawer's filter chips; overlapping text everywhere.
3. **Detail sheet ghosting** — with the full-width DetailPanel open, the drawer + legend + chips remain visible and bleed through the glass (multiple stacked translucent surfaces in one 390px column).
4. **Drawer height** — the drawer (header + always-visible FilterBar) consumes ~55% of the phone screen even "collapsed".
5. **Portrait framing** — the constellation crowds the top half; default camera is tuned for 16:9.
</diagnosis>

<decisions>
## Implementation Decisions (locked) — breakpoint: `max-width: 768px` (and coarse-pointer where behavioral)

### MOB-01 Layout re-flow
- **Title**: compact on mobile — smaller Orbitron size, eyebrow + title tightened; stays top-LEFT but max-width leaves room for the readout.
- **Readout**: compact — smaller figures, tucked top-RIGHT below/next to the title WITHOUT overlap (stack the two figures vertically, reduce padding); or place directly beneath the title bar as a slim strip. NO overlap at 320–768px.
- **Legend**: hidden by default on mobile behind a small "◆ Legend" toggle chip (bottom-left); tapping opens a compact popover; tapping again (or opening any other surface) closes it.
- **Drawer**: on mobile the collapsed state is a SLIM bar (header only — "▲ PIPELINE OVERVIEW"); the FilterBar moves INSIDE the expandable area (not always-visible); expanded ≤ 70dvh, single-column chart stack, internal scroll; use dvh not vh.
- **QR panel**: opens as a CENTERED modal (dark scrim ~55% black) instead of a corner popover; SHARE button stays bottom-right.
- **DetailPanel**: full-width bottom sheet (slide up from bottom on mobile rather than from right), ≤ 85dvh, internal scroll, close ×. Raise its glass background opacity on mobile (e.g. rgba surface ~0.92 or add a solid underlay) so NOTHING ghosts through.

### MOB-02 One primary surface at a time (mobile only)
- A tiny mobile-surface coordinator (CSS `:has()`/class on a shared wrapper, or a small piece of UI state — implementer's choice, but do NOT redesign the runes bridge): when the detail sheet is open → drawer, legend toggle, and SHARE hide (opacity/transform, 150–200ms); when the drawer expands → legend popover closes; when QR modal opens → scrim covers everything else. Closing restores.
- Desktop (>768px) behavior stays EXACTLY as it is today.

### MOB-03 Touch ergonomics
- All interactive chips/segments/buttons ≥44px tap height on mobile (padding bumps, not layout rewrites).
- OrbitControls: confirm one-finger rotate + pinch zoom work (native three behavior) — do not disable touch actions on the canvas (`touch-action: none` on canvas only, page must not scroll-fight the orbit).
- Node tap picking already verified working — don't regress the hit-proxy.

### MOB-04 Portrait camera + perf
- Aspect-aware camera framing in CameraRig: when aspect < 1 (portrait), pull the default (and intro) camera distance back by ~1.35–1.5× (or reduce FOV target) so the full constellation fits; recompute on resize/orientation change. GSAP focus-on-node unchanged.
- Mobile GPU headroom: cap dpr at 1.5 when `matchMedia('(pointer: coarse)')` or width ≤768 (CrystariumCanvas `dpr` prop — currently [1,2]).
- FogExp2/bloom params unchanged (already tuned).

### Hard invariants (regression gate)
- Desktop appearance/behavior unchanged (media-query scoped). layout.js, tokens.ts, derive, runes bridge API, intro API untouched.
- 162 vitest green; `MSYS_NO_PATHCONV=1 BASE_PATH=/Eman_dashboard pnpm build` exit 0; verify-build 6/6; svelte-check clean; no ssr=false; NO new deps.
- Acceptance = orchestrator mobile-viewport Playwright gate (tools/uat-mobile.mjs): no overlap/ghosting on initial view, detail sheet opaque + solo, drawer slim collapsed/scrollable expanded, QR modal centered, tap-select still works, no horizontal scroll, zero console errors. Desktop screenshot must remain unchanged.
</decisions>
