## Style Contract (ui-ux-pro-max)

- **effects-match-style:** dark premium sci-fi. Selective bloom + emissive fresnel crystals over a near-black indigo void. Bloom is the signature, not decoration.
- **dark-mode-pairing:** single opinionated dark theme (PROJECT.md: no light mode). Every surface tier is dark; text is cool-white.
- **blur is purposeful:** `backdrop-filter: blur()` is used ONLY on the glassmorphic HUD panels (legend, title, pipeline readout) to float them over the canvas ‚ÄĒ never on the 3D scene, never decoratively.
- **one primary focus at a time:** exactly one selected node holds primary focus (camera framed + siblings dimmed). Hover is strictly secondary and yields to selection.
- **restraint:** warm color (gold) is scarce by design ‚ÄĒ it marks only the one secured node and the one secured dollar figure, so the eye goes straight to "what is banked."

## Crystarium Layout Contract (CRYS-02, CRYS-04)

Pure module `src/lib/crystarium/layout.js` ‚Üí returns plain `{ nodes:[{id,x,y,z,scale,ring,sector}], edges:[{from,to,kind}] }`. No Three.js import, no RNG (deterministic across builds). Consumes the 28 records from `$lib/data`.

**This contract refines the draft `computePositions` in ARCHITECTURE.md:** the FEATURES/prompt encoding is authoritative ‚ÄĒ **radius encodes status (funnel), angle encodes 501c3 gate.** (ARCHITECTURE's draft had status‚Üíangle; use the reverse below, which is the more legible model.)

### Radial rings (radius = status funnel; center = cash, rim = frontier)

| Ring | Radius (world units) | Status(es) | Count | Members |
|------|----------------------|-----------|-------|---------|
| 0 (core) | 0 | active | 1 | NY Community Trust (master crystal, secured gold) |
| 1 | ~6 | applied | 1 | Brava Thrive Grant |
| 2 | ~11 | in-progress | 3 | Harry S. Black (BofA), Ben & Jerry's, NYC Office for Prevention of Hate Crimes |
| 3 | ~17 | to-research | 17 | Ford √ó2, Third Wave, Giving Joy, NYC Commission, Echoing Green, TGR, Tisch, Kellogg, Yield Giving, BofA Charitable, Wells Fargo, CEK, Milbank, Borealis, SNF, 37 Angels |
| orbit | ~9 (offset inclined orbit) | recurring | 2 | Awesome Foundation, Matriarch Digital Media ‚ÄĒ a slowly-orbiting satellite pair (loop-pulse) |
| dim arc | ~20 (lower/detached band, `y` pushed down) | declined, not-eligible, not-eligible-yet | 4 | Hey Helen (declined), Truist (not-eligible), Just Thrive + TD Bank (not-eligible-yet) ‚ÄĒ a dim "cracked" outer arc, visually fallen off the dome |

### Angular sector (angle within a ring = 501c3 gate)

Within each ring, distribute nodes into three arcs by `requires501c3`:

| Sector arc | `requires501c3` | Meaning |
|-----------|-----------------|---------|
| "Open now" arc (front, őł ‚Čą 200¬į‚Äď340¬į) | `no` | Pursuable today (org is 501c3-pending) |
| "Gated" arc (rear-left, őł ‚Čą 20¬į‚Äď140¬į) | `yes` | Needs 501c3 / fiscal sponsor |
| "Unknown" arc (side, őł ‚Čą 140¬į‚Äď200¬į) | `unknown` | Unverified gate |

Nodes within a sector are ordered by representative amount descending (bigger closer to the sector's leading edge), tie-break by `deadline.date` ascending. Small deterministic ¬Ījitter (`SPREAD`) so a crowded ring fans rather than colinear-stacks.

### Dome wrap

Elevation grows with radius: `y = DOME_CURVE ¬∑ radius + amountBump(node)`. Core sits low-center; the frontier rings curve up into a shallow sphere-cap bowl (the Crystarium dome). `amountBump` gives larger crystals a small extra lift so they read as "taller."

### Node scale range (CRYS-04)

`representativeAmount = amount.avg ?? amount.max ?? amount.min`. Log-scaled:

```
scale = lerp(0.6, 2.4, (log(amt) - log(500)) / (log(200000) - log(500)))   // world-unit radius
```

- Floor `0.6` at $500 (Giving Joy ‚ÄĒ the smallest quantified speck).
- Ceiling `2.4` at ~$200k (Ford JustFilms $125k avg, Ford NYC $100k+ ‚ÄĒ the looming pair).
- **TBD nodes** (`amount.avg/max/min all null` ‚ÄĒ ~13 nodes incl. all "TBD"/"Large"/"Fellowship support" and 37 Angels equity): fixed **0.5** minimal size + the "unformed crystal" material (see Node Visual). They must read as *latent/unquantified*, not as $0.
- NY Community Trust (received $20k) scales normally but its core status treatment dominates regardless of size.

## Path & Beam Contract (CRYS-06)

Three edge kinds in `edges[]`, rendered as `CrystalPath.svelte` (emissive tube/line, additive, low base alpha, animated flow):

### 1. Progression spine (`kind: 'spine'`)
Radial links along the funnel: core ‚Üí ring1 ‚Üí ring2 representatives ‚Üí ring3. Concretely: hub(core) ‚Üí each ring's inward-most node, and node[i]‚Üínode[i+1] within a ring cluster. Color `--path`. This is the "advancement route a funder travels." Steady, low-intensity flow toward the core.

### 2. Funder-family links (`kind: 'family'`) ‚ÄĒ derived from data
Faint bridge edges joining same-funder programs:
- **Ford √ó2:** `ford-foundation-justfilms-documentary-production` ‚ÜĒ `ford-foundation-nyc-good-neighbor-committee`
- **BofA √ó2:** `harry-s-black-allon-fuller-fund-bank-of-america` ‚ÜĒ `bank-of-america-charitable-foundation`

Color `--path` at reduced alpha; thinner than the spine; no directional flow (relatedness, not progression).

### 3. Fiscal-sponsor BEAM (`kind: 'beam'`) ‚ÄĒ the signature (derived from data)
A bright gold‚Üícyan gradient beam from the NY Community Trust core to each funder that the Trust could unlock via fiscal sponsorship. **Target rule:** `requires501c3Raw === 'Yes - or fiscal sponsor'`. Applying that rule to `grants.generated.json` yields **exactly 4 targets**:

|---|---------|------------------|
| 1 | `harry-s-black-allon-fuller-fund-bank-of-america` | Harry S. Black & Allon Fuller Fund (Bank of America) |
| 2 | `ford-foundation-justfilms-documentary-production` | Ford Foundation ‚ÄĒ JustFilms |

## Crystal Node Visual (CRYS-03)

**Geometry direction:** faceted crystal. Base node = elongated hexagonal **bipyramid** (two stacked pyramids ‚ÄĒ a gem shard) OR icosahedron for the core cluster. Low-poly, hard facets (flat-shaded normals) to catch fresnel edge-light. The **core** (NY Community Trust) is a larger multi-shard cluster (3‚Äď5 fused shards) to read as the "master crystal."

**Material direction:** `MeshStandardMaterial` (or Threlte `<T.MeshStandardMaterial>`) with:
- `emissive` = the node's status hue token; `emissiveIntensity` = activation level (below).
- A **fresnel rim** (via `onBeforeCompile` injection or an added rim term) tinted to the status hue ‚ÄĒ gives the translucent crystal edge-glow that reads as "lit from within."
- Moderate `roughness` (~0.25) + slight `transmission`/translucency feel; `metalness` 0. Faceted, glassy, not chrome.

**Status ‚Üí activation level** (facets "light up" as a funder advances ‚ÄĒ CRYS-03):

| Status | `emissiveIntensity` base | Read |
|--------|--------------------------|------|
| to-research | 0.15 | Dim, "unformed" frontier crystal (L1) |
| recurring | 0.20 ‚Üí 0.40 loop | Steady breathing loop (renewable) |
| in-progress | 0.40 | Half-lit, energized (L2) |
| applied | 0.60 | Bright, suspended (L3) |
| active | 1.00 + inner point light | Fully crystallized master (L4) |
| not-eligible-yet | 0.12 + locked bronze halo ring | Lockable-later (gate icon feel) |
| not-eligible | 0.06 | Near-dark, dead |
| declined | 0.06 + subtle "cracked" normal detail | Burnt-out ember |

**Unformed-crystal look (TBD amount):** the ~13 TBD/`isTBD` nodes render at 0.5 scale with a **rougher, less-faceted, more opaque "raw ore" variant** ‚ÄĒ partial facets, higher roughness (~0.5), muted emissive ‚ÄĒ clearly a crystal that "hasn't formed yet" (amount unquantified). 37 Angels (`isEquity`) additionally gets a distinct desaturated/hollow treatment so it never reads as grant money.

## Bloom / Glow Tuning (CRYS-05, CRYS-08)

**Approach:** SelectiveBloom via `postprocessing` (NOT full-scene bloom) ‚ÄĒ only emissive crystal + beam layers bloom; HUD and void do not. Wire through `useThrelte()` renderer + `useTask` composer render, all inside `onMount` (SSR-safe per PITFALLS 1). Perf guards mandatory (PITFALLS 14): `setPixelRatio(min(dpr, 2))`, bloom at **half-resolution**, render-on-demand (pause loop when tab hidden / camera idle).

**UnrealBloom/SelectiveBloom parameter direction (tunable in polish, these are the contract starting points):**

| Param | Value | Why |
|-------|-------|-----|
| `luminanceThreshold` | 0.55‚Äď0.65 | Only bright emissive cores/rims bloom; the dim steel-blue to-research frontier stays crisp and recessive (doesn't wash out). |
| `intensity` / strength | 0.8‚Äď1.2 | Signature glow without haze. |
| `radius` | 0.4‚Äď0.6 | Soft crystalline halo, not a fog. |
| `mipmapBlur` | true, half-res | Mobile-GPU budget. |

**Deadline urgency = pulse (CRYS-05).** Additive `--urgent` coral modulation on `emissiveIntensity`, amplitude + frequency scaled by proximity:

- Pulse **ONLY** when: `deadline.cadence === 'one-time'` AND `deadline.date` is in the future AND `!deadline.isPassed` AND `status ‚ąČ {declined, not-eligible, not-eligible-yet}`.
- Explicitly **no pulse** for: passed (`isPassed`), rolling, annual, invitation, unknown/window cadences, and any declined/ineligible node (design authority: "loops ONLY for non-passed / non-rolling / non-declined nodes").
- Amplitude/speed: near (<30 days) ‚Üí ¬Ī0.4 @ ~0.8 Hz; ‚Č§90 days ‚Üí ¬Ī0.25 @ ~0.5 Hz; farther ‚Üí ¬Ī0.1 @ ~0.3 Hz steady.

**Qualifying nodes today (exactly 3):** Harry S. Black (2026-06-30, in-progress), Ford JustFilms (2026-09-01, to-research), Ben & Jerry's (2027-02-18, in-progress). Hey Helen (2025-12-30, `isPassed`, declined) is a burnt-out ember ‚ÄĒ **never** urgent. The urgency signal is intentionally sparse so it means something.

**Hover vs. selection distinction (CRYS-08):** hover adds a thin fresnel-brighten + small bloom bump (secondary); selection triggers a one-shot activation **burst** (emissive flash spike + scale pop) and raises the selected node's bloom above all siblings while siblings dim ‚ÄĒ an unmistakable primary/secondary hierarchy carried by bloom.

## Camera & Interaction States (CRYS-07, CRYS-08)

State lives in `src/lib/state/crystarium.svelte.js` runes singleton:
```js
export const ui = $state({ selected: null, hovered: null, filter: 'all', cameraFocus: null });
export function select(id) { ui.selected = id; ui.cameraFocus = id; }
export function hover(id) { ui.hovered = id; }
```
**This phase RAISES these events (scene writes on hover/click). Phase 4 CONSUMES them (DetailPanel, filters). Do not build the consumers here.**

| State | Priority | Camera | Node treatment | Timing / easing |
|-------|----------|--------|----------------|-----------------|
| **Idle** | ‚ÄĒ | Slow auto-orbit around the dome (~0.4‚Äď0.8 rad/s equivalent, gentle), slight tilt; `OrbitControls` autoRotate or a GSAP-driven rig. Pauses render-on-demand when truly static is not required (orbit keeps it live) but pause on hidden tab. | All nodes at base activation; recurring pair loop-pulses; ‚Č§3 nodes deadline-pulse. | Continuous, linear-ish drift. |
| **Hover** (secondary) | 2 | No camera change. | Node lifts +Y (small transform), scale +8%, fresnel/emissive +0.2, bloom bump; **billboard label** appears: Orbitron 20/600 funder name + Inter 14/400 sub-line `{repAmount} ¬∑ {daysToDeadline or cadence}` (tabular). | 150ms ease-out in; ~120ms out (exit faster than enter). |
| **Selected** (primary ‚ÄĒ one at a time) | 1 | Camera **springs** to frame the node (GSAP ease-out, ~550‚Äď650ms), dolly + look-at the node; shallow depth-of-field feel via sibling dimming. | Selected plays activation **burst** (emissive flash + scale pop, ~300ms); ALL other nodes dim (emissiveIntensity √ó0.35 + slight desaturate) and drop bloom. Writes `ui.selected` + `ui.cameraFocus`. | Enter ~600ms ease-out; **deselect/exit ~250ms** (faster than enter). |

**Motion discipline (design authority):** all animation is transform / opacity / material-uniform driven only (position, scale, emissiveIntensity, fresnel power). Micro-interactions 150‚Äď300ms; camera focus is a spring/ease-out; every exit is faster than its enter. No layout-thrashing, no continuous full-scene re-render when idle-and-hidden.
