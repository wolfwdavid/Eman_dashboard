# Phase 4 — UI Design Contract · HUD / Overlay UI (+ minimal fallback)

> Visual and interaction contract for the 2D DOM dashboard layered over the fixed Threlte Crystarium canvas. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> **This phase EXTENDS `03-UI-SPEC.md`.** Every token (color, type, spacing, glass style, motion discipline) is inherited verbatim — the overlay must feel like the SAME product as the scene. ## Layout & Layer (z-index) Model — the Crystarium stays the hero

The Threlte `<Canvas>` is a **fixed, full-viewport, interactive** hero at the back. Every overlay piece is a **discrete `position: fixed` element** — there is **NO full-viewport catch layer** (a full-screen overlay `div` would eat the canvas raycast and kill orbit/hover/select). Interactive panels set `pointer-events: auto` on themselves only; the ambient Phase-3 HUD stays `pointer-events: none`. Empty overlay space passes clicks straight through to the scene.

| Layer | z-index | Contents | pointer-events | Owner |
|-------|---------|----------|----------------|-------|
| Canvas (hero) | `0` | Threlte Crystarium, full viewport, fixed | auto (raycast) | Phase 3 |
| Ambient HUD | `10` | SceneTitle (top-left), PipelineReadout (top-right), Legend (bottom-left) | **none** | Phase 3 (unchanged) |
| Interactive controls | `20` | Filter bar + Pipeline Overview drawer (bottom-center), QR toggle widget (bottom-right) | auto | Phase 4 |
| Detail Panel | `30` | Right-edge slide-in rail (opens on `ui.selected`) | auto | Phase 4 |
| QR panel (expanded) | `30` | Glass panel above its bottom-right toggle | auto | Phase 4 |

**Corner map (desktop, edge inset `lg`/24px):**

```
┌ TL: Title (P3) ──────────────────── TR: Pipeline Readout (P3) ┐
│                                                                │
│                    ◇  CRYSTARIUM CANVAS  ◇        ┌───────────┐│
│                        (hero, fixed)             │  DETAIL   ││
│                                                  │  PANEL    ││ ← z30, slides in
│                                                  │ (on select)│
│ BL: Status Legend (P3)                           └───────────┘│
│         ┌──────── BC: Filter bar / Pipeline drawer ─┐  QR ◹ BR │
└─────────┴──────────────────────────────────────────┴──────────┘
```

- **Detail Panel** occupies the right edge (full height, ~380px). When open it overlays the top-right Readout and bottom-right QR toggle (opaque glass, z30) — that is acceptable: selection is a temporary focused state and the scene stays live behind it.
- **Pipeline Overview** is a **bottom-center collapsible drawer**. Collapsed by default (a handle + the persistent Filter bar) so the charts never fight the scene. Expanding slides the 4 charts up from the bottom edge.
- **QR toggle** is a small **bottom-right** widget (the one free corner). Expands upward into a 2-tile QR panel.
- **One primary focus (design authority):** the open Detail Panel is the single primary focus while a node is selected — it mirrors the scene's "one selected node." Overview and QR are secondary toggles and yield to it visually.

**Responsive (minimal — not an a11y requirement):** below 640px the Detail Panel becomes a full-width bottom sheet, the Pipeline drawer spans full width, the QR panel centers, and all edge insets collapse to `md`/16px.

## Detail Panel (DETL-01 / DETL-02 / DETL-03)

Right-edge slide-in glass rail (`--surface-glass` + `blur(16px)` + `--surface-glass-border`, radius 14px, padding `lg`/24px, width ~380px). Opens when `ui.selected != null`; looks up the record with `grants.find(g => g.id === ui.selected)`. **Surfaces all 9 `Grant` fields**, shaped. Scrolls internally if content overflows.

**Header**
- Node-hue accent: a 12×12px crystal swatch in `var(--status-{status})` + a hairline top-border in the same hue (echoes the selected crystal — spatial continuity).
- Funder title: `grant.funder` — Orbitron 20/600, `--text-hi`.
- Program subtitle (if `grant.program`): Inter 14/400, `--text-lo`.
- Close affordance: `×` button top-right (Inter, `--text-lo` → `--text-hi` on hover), calls `deselect()`.

**Field rows** (each: Inter 12/400 uppercase eyebrow in `--text-lo` + value below)

| # | Field | Eyebrow | Presentation |

## Filter Controls (PIPE-05) — persistent, in the drawer header

Always visible (even when the drawer is collapsed) so segmenting is one click. Three axes; each writes the structured filter (below). Active filters are visually indicated; a `Reset` clears all.

| Axis | Control | Values | Writes |
|------|---------|--------|--------|
| Status | Chip row (single-isolate) — each chip = hue swatch + label + count | All / the 8 statuses | `ui.filter.status` |
| 501(c)(3) | Segmented control | All / Open / Gated / Unknown | `ui.filter.gate` |
| Type | Segmented control | All / Grant / Fellowship / Investment | `ui.filter.type` |

Chip/segment interactions animate 120ms (color/opacity only). Active chip = full hue fill; inactive = hue swatch + `--text-lo` label on glass.

**Zero-match empty state:** if the active filter combination matches no funders → charts + scene show the empty message *"No funders match these filters."* + a `Reset filters` action.

## Scene Reflection of Filters (PIPE-05)

Filters **dim** (do not delete) non-matching nodes so the layout stays stable and the funnel shape is preserved.

- A node **matches** when it satisfies all active axes: `(status==='all' || g.status===status) && (gate==='all' || gateBucket(g)===gate) && (type==='all' || typeBucket(g)===type)`.
  - `gateBucket`: `requires501c3 'no' → 'open'`, `'yes' → 'gated'`, `'unknown' → 'unknown'`.
  - `typeBucket`: `'Grant' → 'Grant'`, `'Grant/Fellowship' → 'Fellowship'`, `'Investment' → 'Investment'`.
- **Non-matching** nodes: `emissiveIntensity ×0.15`, mesh opacity ~0.3, raycast/pointer **disabled** (can't hover/select a filtered-out node); their spine/family/beam edges dim in step.
- **Matching** nodes keep full treatment. Transition ~200ms (material-uniform + opacity only), consistent with Phase-3 motion discipline.
- OPTIONAL (nice-to-have, not required): the top-right Pipeline Readout MAY recompute secured/potential for the active subset. The **required** behavior is the scene dim.

## QR Panel (QRUI-01 / QRUI-02)

Bottom-right toggle widget (the free corner). A small glass button `SHARE ◹` (Orbitron 14) expands **upward** into a glass panel titled `SHARE` (Orbitron 20/600) holding **two QR tiles side by side** (gap `xl`/32px).

Data source: `import { qrCodes } from '$lib/data'` — an array of `{ id, label, url, svg }` (inline SVG, build-time generated, no runtime fetch). Render `{@html qrCodes[i].svg}` inside each tile.

**Each tile:**
- **White plate** (rounded, `md`/16px padding) holding the inline QR SVG — QR codes REQUIRE a light background for scannability; the plate is intentionally the one white surface in the dark theme. Plate size clamps 112–160px square.
- Label below: `qrCodes[i].label` (`Visit DID`, `Support / Tracker`) — Inter 14/400 `--text-hi`.
- URL hint: `qrCodes[i].url` — Inter 12/400 `--text-lo`, truncated with ellipsis.

**Config-swap note (QRUI-02):** the two URLs live ONLY in `src/lib/config/sites.js`. Editing them and re-running `node tools/generate-qr.mjs` (or `pnpm build`) regenerates `src/lib/data/qr.generated.js`. The panel imports `qrCodes` and needs **zero code changes**. The QR encodes the ABSOLUTE external URL (never the `base`-prefixed dashboard route). NOTE: the `support` URL is currently a `REPLACE-ME` placeholder — the tile renders regardless; the placeholder is swapped at config time, not in the component.

Toggle animates from the button (transform-origin bottom-right, scale+opacity, 200ms enter / 140ms exit).

## Fallback — Minimal Graceful Degradation (RESL-01 · OPTIONAL · LOWER PRIORITY)

> **Priority:** clearly below DETL / PIPE / QRUI. RESL-01 is a **v2** requirement. Ship only a *minimal* no-black-screen degradation — NOT a full parallel UI, NOT a11y.

- **Detection:** in `onMount`, probe `document.createElement('canvas').getContext('webgl2') || getContext('webgl')` (and/or catch Threlte canvas init failure). If unavailable, do not mount `<Canvas>`.
- **What still works for free:** the entire overlay is DOM + data-driven, so the Title, Readout, Legend, Detail Panel, Pipeline Overview (charts + filters), and QR panel **all render without WebGL**. They read the same `grants` JSON.
- **Fallback add:** in place of the canvas, render a **simple 2D grant list** from the same `grants` array — a scrollable list of the 28 funders, each row: funder name · status pill (hue) · human-readable amount · deadline chip · `Open ↗` link. Clicking a row calls `select(id)` so the existing Detail Panel opens (no new detail UI needed).
- **Copy:** a one-line notice at top of the list — *"3D view unavailable on this device — showing the grant list."*
- **Reuses:** the amount/deadline/gate display helpers from the Detail Panel; the status pill from the Legend hues. No new tokens, no new state.

## Interaction & State Contract (the Phase-3 → Phase-4 bridge)

Phase 3 **raises** events into the `crystarium.svelte.js` runes singleton; Phase 4 **consumes** them.

**State extension (planner note):** `ui.filter` must be widened from the current single string to a structured object, plus setters. This is the one required change to the shared state module:

```js
// crystarium.svelte.js — extend:
export const ui = $state({
  selected: null, hovered: null, cameraFocus: null,
  filter: { status: 'all', gate: 'all', type: 'all' }   // was: 'all'
});
export function setFilter(axis, value) { ui.filter[axis] = value; }
export function resetFilters() { ui.filter = { status: 'all', gate: 'all', type: 'all' }; }
// select() / deselect() / hover() unchanged.
```

| Trigger | Reads/Writes | Result |
|---------|--------------|--------|
| Scene raycast click → `select(id)` | writes `ui.selected`, `ui.cameraFocus` | Detail Panel slides in for that grant; camera frames the node (Phase-3 behavior) |
| Detail `×` / `Esc` / click scene background → `deselect()` | clears `ui.selected`, `ui.cameraFocus` | Panel slides out; camera releases to idle orbit; overview restored |
| Filter chip/segment → `setFilter(axis,val)` / `resetFilters()` | writes `ui.filter.*` | Scene dims non-matching nodes; charts + list reflect the subset |
| QR `SHARE` toggle | LOCAL component `$state` (NOT the shared `ui`) | QR panel expands/collapses |
| Fallback list row click → `select(id)` | writes `ui.selected` | Detail Panel opens (same path as a node click) |

**Close affordance (required):** every focused panel returns to overview — Detail Panel via `×`/`Esc`/background-click; Pipeline drawer via its handle; QR via its toggle.
